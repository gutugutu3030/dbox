# AGENTS.md

このファイルは AI エージェント（opencode 等）がこのリポジトリでコードを編集する際の規則を定義します。

## コメント規則

- コメントは **日本語** で記載すること
- コメントは **関数単位** で記載すること（関数の宣言直上に配置）
- 関数の長さが **100行を超える場合** は、関数内にも処理ブロック単位でコメントを挟むこと
- `TODO` コメントは `TODO: <内容>` の形式で記載し、担当者と課題を明確にすること

### 良い例

```go
// LoadProjectConfig はプロジェクト設定ファイルを読み込む。
// ファイルが存在しない場合は ErrNotExist を返す
func LoadProjectConfig(dir string) (*ProjectConfig, error) {
```

### 悪い例

```go
func LoadProjectConfig(dir string) (*ProjectConfig, error) {  // 設定読み込み
```

## ファイルサイズ規則

- 1ファイルの最大行数は **400行** とする
- 400行を超える場合は、適切な責務でファイル分割すること
- 分割の単位は「責任」（例: コマンド定義、ビジネスロジック、入出力）とすること

## テスト規則

- すべての公開関数・メソッドには **単体テスト（ユニットテスト）** を作成すること
- テストファイルは対象ファイルと同じパッケージ、同じディレクトリに配置すること
- テストファイル名は `{対象ファイル名}_test.go` とすること
- テーブル駆動テストを推奨する
- `t.Cleanup` を活用してテストの後処理を確実に行うこと

## コードスタイル

- Go 標準のフォーマッタ（`gofmt` / `go fmt`）に従うこと
- 未使用の import や変数は削除すること（`go vet` で警告が出ない状態を維持）
- エラーハンドリングは必ず行い、エラーを無視しないこと
- エラーメッセージは日本語で記述すること
